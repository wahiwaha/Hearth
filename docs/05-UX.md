## 4.1. 사진 업로드까지의 경험

### 사진 저장

- 개인이 사진을 찍으면, **자동으로 앱 속 갤러리에도 저장됨** (접근권한 필수)

### 앨범 생성

- 라이브러리에 가서 빈 페이지의 앨범을 만듦
- 처음 만들어진 앨범은 **3페이지** (커버 2페이지, 본문 1페이지)

### 페이지 관리

- 굿노트처럼 각 페이지들을 볼 수 있는 버튼을 누르면 전체 페이지가 보임
- 버튼을 눌렀을 때의 페이지를 기준으로 **앞/뒤에 페이지 추가** 가능

### 그리드 시스템

- 기본 생성된 본문 폴리오 및 새로 생성한 본문 폴리오에는 **그리드를 설정**할 수 있음
- 앨범 만드는 앱들처럼 사진 배치를 예쁘게 할 수 있도록 **추천 틀**이 있음
- 이 틀을 실제로 불러오고, 틀 속에서 사진이 들어가는 위치를 **직접 수정**할 수 있음
- 틀을 그대로 따를 수도 있음
- **틀을 안 불러오고 개인이 사진을 막 배치하는 것도 가능**

### 사진 배치 방법

1. **틀 기반:** 틀을 불러와서 지정된 사진 넣는 곳에 사진을 불러와서 어느 부분을 담을 것인지만 설정
2. **자유 배치:** 사진 자체를 불러와서 알아서 배치

### 페이지 꾸미기 요소

- 폴리오에 들어갈 **기본 배경**
- 각종 **이모지**
- **그림** 등을 추가하여 앨범을 꾸밀 수 있음
- 사진을 불러온 다음에 **원하는 각도, 원하는 크기**로 사진을 배치 가능
- **두 손가락 드래그**로 다 조정 가능

### 사진 메모

- 사진에 메모를 할 수도 있음
- 이 메모는:
  - **이 사진에 항상 적용** (글로벌 메모)
  - **이 앨범 내에서만 적용** (앨범 로컬 메모)
  - 두 가지 중 선택 가능

## 4.2. 꾸미기 경험

사용자가 앨범을 다 불러오고 사진도 원하는 곳에 배치했다면, 그때부터는 앨범 및 사진을 꾸며야 함.

### 사용 가능한 꾸미기 방법

- 앱이 제공하는 **멋진 배경** 사용
- 따로 **배경사진 불러오기**
- **그림 그리기**
- 사진을 액자처럼 **사진 자체에 해당하는 디자인 적용**
- 다양함. **상상하는 모든 걸 할 수 있어야 함**

### 예시

- 사진 후보정 (필터)
- 그림 그리면 **AI가 있어보이는 그림으로 바꿔줌**
- 등등

## 4.3. 공유 경험

사용자가 앨범 설정을 누르면, 세 가지 공유 옵션이 있음:

### 옵션 1: 앨범 전체 공유

- 앨범 커버, 폴리오, 앨범 속 모든 사진이 공유됨
- 다른 사람에게 공개되는 **기본 범위는 사진 한 장**
- 소셜에서 설명한 것처럼, 인스타처럼 사진을 눌러 정보를 확인
- **Hearth만의 차별점:**
  - 사진을 **축소하면** 사진이 속한 페이지가 보임
  - 더 **축소/좌우 스크롤**하면 다른 페이지와 앨범 커버 등을 다 볼 수 있음
- 검색할 때 필터에 폴리오/커버를 택하면 해당 콘텐츠가 보임

### 옵션 2: 선택한 페이지 공유

- 선택한 페이지가 공유됨
- 선택한 페이지 속 사진들이 다른 사람들에게 보임
- 다른 사람들은 그 사진을 눌러 게시자의 앨범을 볼 수 있음
- **특정 사진 가리기:**
  - 선택한 페이지 공유를 누른 다음, **특정 사진 가리기**를 눌러 페이지 속 블러처리할 사진을 고를 수 있음
  - 이 사진은 타인의 탐색 피드에 나타나지 않음
  - 공개된 다른 사진을 통해 앨범을 볼 때도 **블러처리**되어 보이지 않음

### 옵션 3: 선택한 사진 공유

- 이것을 누르면 폴리오를 택할 때와는 달리 해당 앨범 속 사진이 **갤러리처럼 목록화**되어 보임
- 여기서 공유할 사진을 선택함
- 인스타와 유사
