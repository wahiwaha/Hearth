# 19. 현실적 로드맵

> 최종 수정: 2026-03-04
>
> *11-로드맵.md의 이상적 계획을 솔로 개발자 + Claude Code 바이브코딩 기반으로 재구성한 실행 가능한 버전.*

---

## 전제 조건

- **개발자:** 1인 (오찬영) + Claude Code
- **풀타임 기준**, 하루 8-10시간 투입 가정
- **투자금 없음** — 자비 또는 소액 부트스트래핑
- **기술 스택:** React Native (Expo) + Firebase + react-native-skia
- **스프린트:** 2주 빌드 + 3일 회고/버그픽스

---

## Sprint 0 — 환경 세팅 & 디자인 시스템 (2주)

**기간:** 2026년 3월

| 할 일 | 산출물 |
|--------|--------|
| Expo 프로젝트 초기화 | 빌드 가능한 빈 앱 |
| Firebase 프로젝트 생성 및 연결 | Auth, Firestore, Storage 연결 확인 |
| 디자인 토큰 정의 (컬러, 타이포, 스페이싱) | tokens.ts |
| 핵심 공통 컴포넌트 (Button, Input, Card, Modal) | 컴포넌트 라이브러리 v0 |
| 네비게이션 구조 설계 (탭, 스택) | 앱 뼈대 |

**완료 기준:** `expo start`로 앱이 뜨고, 빈 탭 네비게이션이 작동

## Sprint 1-2 — 에디터 코어 프로토타입 (4주)

**기간:** 2026년 3월 말 ~ 4월

> 가장 중요한 단계. 여기서 기술적 가능성을 검증 못 하면 피봇 필요.

| 할 일 | 산출물 |
|--------|--------|
| react-native-skia 세팅 및 기본 캔버스 | 빈 캔버스 렌더링 |
| 사진 선택 → 캔버스 배치 | 사진 1장을 캔버스에 올리기 |
| 드래그, 핀치 줌, 회전 제스처 | 사진 조작 가능 |
| 텍스트 오버레이 추가/편집 | 캔버스 위에 텍스트 |
| 스티커 5-10종 추가 기능 | 스티커 선택 → 배치 |
| 캔버스 상태 JSON 직렬화/역직렬화 | 저장/불러오기 가능 |

**완료 기준:** 사진 1장 + 텍스트 + 스티커를 배치하고 저장/불러오기가 되는 프로토타입

**킬 스위치:** 4주 후에도 기본 제스처가 안정적이지 않으면 → 대안 검토 (웹뷰 기반 에디터, 또는 더 단순한 그리드 레이아웃 방식으로 전환)

## Sprint 3-4 — 개인 앨범 MVP (4주)

**기간:** 2026년 5월

| 할 일 | 산출물 |
|--------|--------|
| Firebase Auth 연동 (Apple/Google 소셜 로그인) | 로그인/회원가입 플로우 |
| Firestore 앨범 스키마 설계 | 앨범 CRUD |
| 앨범 생성 → 사진 선택 → 에디터 → 저장 풀 플로우 | 핵심 유저 저니 완성 |
| 앨범 리스트 (홈 화면) | 내 앨범 목록 |
| 사진 업로드 → Firebase Storage | 클라우드 저장 |
| 앨범 커버 자동 생성 (첫 번째 사진 기반) | 앨범 썸네일 |

**완료 기준:** 가입 → 앨범 만들기 → 사진 꾸미기 → 저장 → 다시 열기까지 끊김 없이 작동
