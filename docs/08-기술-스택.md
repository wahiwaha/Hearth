# 08. 기술 스택

> 최종 수정: 2026-03-02

---

## 최종 선택: React Native (Expo) + Firebase + react-native-skia

### 선택 이유

- **Expo:** bare RN 대비 바이브코딩 속도 2배, EAS Build로 앱스토어 배포까지 Claude Code 처리 가능
- **Firebase:** 기존 바이브코딩 경험 있음, React Native SDK 최적화 완료
- **react-native-skia:** 인스타 스토리 수준의 꾸미기 편집기를 구현할 수 있는 유일한 현실적 옵션
- **Claude Code:** React 생태계를 압도적으로 잘 알고 있음

---

## 핵심 기술적 난제: 꾸미기 에디터

Skia 기반 에디터 설계를 초반에 잘못 잡으면 나중에 전체를 갈아엎어야 할 수 있다. 따라서 **Phase 1의 핵심**.

---

## 얼굴 인식 기술

- 얼굴 인식 기술 자체는 새로운 게 아님. 애플 사진 앱, 갤러리가 이미 하고 있는 것
- Hearth의 차별점은 인식 *다음*에 오는 행동 — "이 사람과 앨범 만들래요?"라는 유도
- **구현:** Google ML Kit 얼굴 인식

---

## AI API 통합 계획

| AI | 용도 |
|----|------|
| **Google ML Kit** | 온디바이스 얼굴 인식 → "이 사람과 앨범 만들래요?" 자동 제안. 서버 미전송으로 GDPR/개인정보보호법 준수 용이 |
| **Claude API (Anthropic)** | 자동 캡션 생성, 레이아웃 추천, "어제 술자리 사진들로 앨범 꾸며줘" 같은 자연어 지시 처리 |
| **GPT API (OpenAI)** | 필요시 Claude API 보조 또는 대안으로 검토 |

> AI 기능은 출시 후 리텐션 지표 확인 뒤 도입 (M3 단계). 초기 MVP에는 포함하지 않는다.
